Method

The UV grid is the data structure

Visibility-driven UV sampling

Per-splat visibility from the rasterizer's out_observe. Every $K$ iterations, UV samples are re-placed by inverse-CDF of the visibility marginals — dense where cameras look, sparse where occluded. Constant splat budget, zero overhead.

bounded UV — no boundary degeneracy
$u=\epsilon+(1-2\epsilon)\varphi(\tilde u),\ \ \epsilon=0.01$

Immune to the Swiss-Roll problem

UV adjacency indexes geodesic neighbors. Points that are Euclidean-close but manifold-distant never become spurious neighbors — P[i±1, j] is structurally on-surface. Smoothness and coverage losses collapse to stride-1 array reads.

Meshing / neighborhood cost — n = splat count
O(V), V≫n Marching Cubes / Poisson

voxel grid over the whole bounding box · needs normals

O(n log n) Delaunay / Ball-pivoting

KD-tree lookups · unstructured output

O(n) · O(1) neighbors UV kernel scan (ours)

the mesh is the UV grid — no search, no hyperparameter

Every mesh, neighbor query, and regularizer is an array operation on the parametric grid.
Resolution

Fixed budget, infinite resolution

The surface is continuous — splat count is a render-time sampling choice, not a property of the model.

Zoom in — subdivide the UV crop

Restrict $(u,v)$ to the region of interest, keep the same $S\times S$ sample grid: each splat's geodesic footprint contracts. Detail rises at constant cost.

Zoom out — coarsen for free

Same surface rendered with fewer, larger splats for far views — a principled LOD ladder with no retraining and no extra storage.

A trained 3DGS model can do neither — its particle count is baked in at training time.

Emergent capability

3D segmentation through the 2D feature domain

Every Gaussian lives at a UV coordinate — a click in 2D feature space is already a view-independent 3D segment.

① click in feature space

A distilled feature field (DINO/CLIP) lives on the same UV grid. One click → thresholded similarity mask. No 3D clustering.

② lifts to 3D — roof

The UV mask selects exactly the Gaussians on that surface patch — coherent from views never used for the click.

③ complement — walls

Everything not selected is an equally sharp complementary region — same UV domain, one boundary.

Why 3DGS can't do this cleanly: without a shared 2D domain, features must be clustered in ℝ³ — noisy, view-inconsistent, blind to surface adjacency. Here, “segment” and “UV region” are the same object.

Analysis

Each design choice earns its place

Multilevel B-spline initialization

Random init: geometry losses meaningless until ~iter 7k. Multilevel B-spline prior: active at iter 150 — 50× earlier, and the only reason early normal supervision works at all.

Visibility-driven UV sampling

Uniform grid wastes splat budget on occluded UV regions. Inverse-CDF re-placement keeps the budget where cameras actually look — at zero memory overhead.

Adaptive knot insertion

Fixed knot grid under-resolves high-curvature regions. Error-driven Boehm insertion tracks curvature demand; exactness means densification never destabilizes training.

NURBS weights + residual scaling

Next experiment. Both disabled in the 18.3 dB ablation. Full config adds conic precision and per-splat opacity range — the hypothesis: geometry doesn't need them, photometric quality does.

18.3 dB is the floor — 1/20th training, two features off. The ablation order makes explicit what each knob controls.
Limitations

What this doesn't do yet — and what's next

Known & disclosed

Single-patch topology

One B-spline patch per surface component, fixed before training. A hole, or two touching objects, isn't representable — the multi-surface extension is future work, not solved today.

Conclusion

Summary & future work

Contributions

  • One learnable NURBS surface replaces millions of unstructured Gaussians — splats derived, not stored
  • Differential geometry supplies rotation & scale with zero parameters
  • Multilevel B-spline init activates geometric supervision 50× earlier
  • Decoupled densification: exact knot insertion + visibility-driven UV sampling
  • Fused CUDA surface-evaluation kernel — ~30× over einsum
  • Post-training editability, LOD, CAD export — by construction

Future work

  • Full 30k convergence + sweep on all DTU scans
  • Fold surface evaluation into the rasterizer (end-to-end fusion)
  • Multi-surface scenes; holes and genus > 0
  • NURBS weight optimization for sharp features
  • Swarm-style active-view resolution control (as simulated)

Core thesis: structure is not a constraint — it's free information. A surface knows its own normals, scales, and orientations.
